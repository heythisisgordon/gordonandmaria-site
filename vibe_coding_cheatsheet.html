Start Vibe Coding
Use a structured approach to vibe coding that mirrors human-centered design processes.
üéµ The Vibe Coding Philosophy
You're the Architect Structure work in Plan mode, let LLM handle everything in Act mode. Never type code yourself.
Continuous Development Loop Continuous Plan ‚Üí Act ‚Üí Assess development loop mirrors the human-centered design process.
Full Dev Team Let the LLM form a complete cast of characters. You manage and direct the team.

üß† Plan Mode

  • LLM reads all your code
  • Cannot edit files (usually)
  • Has extended thinking enabled
  • End prompts with "Plan how to..."

üöÄ Act Mode

  • Receives the plan + context
  • Can edit and create files
  • Executes the planned actions
  • Faster, focused execution
5
Use MCP Servers for Even More Structured Development

üìã Manage Issues

  • LLM generates and tests initial code
  • Identifies areas for improvement
  • Automatically creates GitHub Issues
  • Documents enhancement requirements

üéØ Prioritize Tasks

  • GitHub MCP server analyzes all issues
  • Ranks by importance and impact
  • Considers dependencies and complexity
  • Selects highest priority next task

üîÑ Iterate

  • Address the selected priority issue
  • Implement, test, and validate solution
  • Close completed issue and update repo
  • Move to next highest priority task
üîß What are MCP Servers?
Model Context Protocol (MCP) servers extend your LLM's capabilities by connecting to external tools and services. They act as bridges between Claude and your development workflow.
üéØ Structured Development Benefits
This approach creates a self-improving development loop where your AI assistant continuously refines and enhances your codebase through systematic issue tracking.
